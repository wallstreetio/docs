# proofread

## Instructions 


* Starting at a random spot, go through the documentation until you find 50 typos or finish reading the documentation. 
* Only look for typos. Don't propose any major changes. 
* Make the changes for the typos. 
* Summarize the changes in a numbered list by file and highlighting only the change.
* 

## Extra Guidance
* Wallstreet.io is Wallstreet.io not the Wallstreet.io
* Don't use you, use the user.
* Don't run another command line spell checker. There is already a pre-commit hook for one.
